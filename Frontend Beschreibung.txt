Frontend Beschreibung

CITY WALLET — Detaillierte Frontend-Spezifikation
Für deinen Freund, der den Backend und die 3D-Karte gebaut hat. Diese Spec geht davon aus, dass beide bereits existieren und das Frontend darauf aufsetzt. Sie ist so geschrieben, dass er sie direkt umsetzen oder an ein Code-Gen-Tool (Cursor, v0, Lovable, Claude Code) weitergeben kann.

1. KONTEXT & ZIELSETZUNG
Was wir bauen: Eine mobile-first Web-App namens CITY WALLET — ein KI-gestützter, kontextbewusster Stadtbegleiter, der hyperpersonalisierte lokale Angebote in Echtzeit anzeigt. Hackathon-Prototyp für die DSV-Gruppe Challenge.
Was bereits existiert (von deinem Freund gebaut):

Backend mit Mock-API-Endpoints für Kontext-Signale, Angebote, User-Daten
Eine 3D-Karte von Darmstadt (Style: axonometrisch / isometrisch, vermutlich Three.js, Spline oder Mapbox 3D)

Was das Frontend leisten muss: Die 3D-Karte und die Backend-Daten in ein emotional aufgeladenes, atmosphärisches UI einbetten, das in einem 90-Sekunden Demo-Video überzeugt.
Demo-Szenario (alle Mock-Daten müssen darauf abgestimmt sein):

User: Amine, 28
Stadt: Darmstadt
Tag: Sonntag (morgen)
Zeit im Demo: 13:47
Wetter: 31°C, sonnig, hoch UV
Standort: Luisenplatz
Hero-Merchant: BACK FACTORY, Luisenplatz
Hero-Angebot: Kalte Apfelschorle + frische Brezel — €3,50 statt €5,20


2. TECH-STACK (Empfehlung)

Framework: Next.js 14+ (App Router) oder Vite + React 18
Styling: Tailwind CSS + shadcn/ui als Basis, stark customized
Animation: Framer Motion (Pflicht — die App lebt von Motion)
State: Zustand oder React Context (kein Redux nötig)
3D-Karten-Integration: Je nachdem was dein Freund gebaut hat — als React-Komponente einbinden, ggf. via iframe oder direkter Three.js Mount
Fonts: Geist + Geist Mono (sans/mono) und Instrument Serif (display) — alle via next/font oder Google Fonts
Icons: Lucide React
Viewport-Target: 390px breit (iPhone 14/15), aber responsive bis Desktop für die Demo-Aufnahme


3. ÄSTHETISCHE RICHTUNG — "AMBIENT OS"
Das ist die wichtigste Sektion. Wenn dein Freund nur eine Sektion liest, dann diese.
Grundgedanke: Die App soll sich anfühlen wie die Stadt atmet um den User herum. Nicht wie eine Fintech-App. Nicht wie eine Coupon-App. Mehr wie eine Wetter-App, die zufällig Angebote zeigt.
Referenzen: Apple Weather (Redesign 2023+), Mercury Weather, Raycast, Things 3, Arc Browser. Nicht Stripe, Revolut, Lidl Plus, Payback.
Hintergrund als Stimmung
Der gesamte App-Hintergrund ist ein Full-Bleed Gradient, der das aktuelle Wetter und die Tageszeit darstellt. Für unser Demo-Szenario (heißer Sonntagnachmittag):

Gradient-Stops von oben nach unten:

#FFE8D1 (warmes goldenes Pfirsich)
#F9C9A8 (apricot midtone)
#E8B5C4 (dusty rose)
#C9A6BD (rose-lavender)


Subtiles volumetrisches Sonnenlicht in der oberen rechten Ecke (radialer Gradient von #FFFAEB mit 40% Opazität, blur 80px), driftet langsam
Hitzeflimmern-Partikel: kleine helle Punkte (4-8px), 1-3% Opazität, langsam aufwärts treibend (CSS-Animation, 20-30s Dauer pro Partikel)
Der Gradient atmet: 12-Sekunden-Loop, Sättigung shiftet um 3-5%
Optional: 1-2% film grain Overlay für Textur

Farbpalette
VerwendungFarbeHexPrimärtextwarmes Schwarz#1A1410Sekundärtextwarmes Braun, 70%#5C4A3FTertiärtextgedämpft#8A7268Card SurfaceGlasrgba(255, 255, 255, 0.08) mit backdrop-blur-2xlCard BorderHairlinergba(255, 255, 255, 0.18)Primärakzent (sparsam!)warmes Amber#E8893BBestätigung (einmalig)dusty green#7BA87A
Verbotene Farben: Reines Weiß, reines Schwarz, gesättigtes Blau, gesättigtes Rot, Neon-Töne. Wenn etwas blau ist, ist es falsch.
Typografie

UI-Body: Geist Sans, 14-15px, line-height 1.5
UI-Mono / Daten / Timestamps: Geist Mono, 11-13px, letter-spacing 0.02em
Display / emotionale Headlines: Instrument Serif, 28-44px, line-height 1.1, weight 400
Hierarchie-Regel: Mono = System spricht. Sans = App spricht. Serif = App fühlt.

Layout-Philosophie

Single column, mobile-first
Großzügige vertikale Abstände (24-48px zwischen Sektionen)
Cards sind Andeutungen von Containern, keine Boxen — sehr niedriger Kontrast
Bottom-Navigation als floating glass pill, nicht als Bar
Die 3D-Karte ist ambient backdrop, nicht der Held — sie darf nicht mit dem Angebots-Card konkurrieren

Motion-Prinzipien

Default Easing: cubic-bezier(0.32, 0.72, 0, 1)
Default Duration: 400-800ms
Bewegungssprache: Dinge blühen auf und setzen sich — sie sliden oder poppen nicht
Standard-Entry: scale 0.94 → 1.0, opacity 0 → 1, y-translate 8px → 0
Idle Motion überall:

Gradient atmet (12s loop)
Sonnen-Glare driftet (40s loop)
Moment Card Pulse am warmen Rand (4s loop)
User-Dot auf Karte: context aura pulse (2s loop)


Wenn etwas schnell wirkt, ist es kaputt. Maximale Geschwindigkeit ist sanft.


4. KOMPONENTEN-ARCHITEKTUR
Diese Komponenten bauen, jeweils mit shadcn/ui als Basis wo möglich:
Atomare Komponenten
<GlassCard />
Wrapper für alle Cards. Props: variant (subtle | elevated | hero), className. Implementiert: backdrop-blur, hairline border, soft ambient shadow, 1% noise overlay. Alle anderen Cards extenden diese.
<ContextChip />
Kleiner Mono-Chip mit Icon links und Label rechts. Verwendet im ContextStrip und Why-Reveal. Höhe 24px, padding 6px 10px, rounded-full, border hairline.
<AmberButton />
Primärer CTA. Solid Amber #E8893B, weißer Text, rounded-full, padding 16px 24px. Hover: scale 1.02, brightness 1.05. Tap: scale 0.98. Soft glow shadow 0 8px 24px rgba(232, 137, 59, 0.3).
<GhostButton />
Sekundärer CTA. Transparent, warmer Sekundärtext, kein Border. Hover: 5% white background.
<SerifHeadline />
Display-Komponente für emotionale Momente. Instrument Serif, props: size (sm 28px | md 36px | lg 44px).
Composite Komponenten
<ContextStrip /> (top of home screen)
Horizontale Strip ganz oben. Mono-Schrift, sehr klein (11px). Inhalt:
Darmstadt · 13:47 · 31° ☀ · Sonntag · 3.2 km/h
Trennzeichen: · mit reduzierter Opazität. Live-feel: Zeit tickt jede Sekunde, Walking-Pace fluctuiert ±0.2 km/h alle 5 Sekunden.

<AmbientMap /> (Integration der 3D-Karte deines Freundes)
Container der die 3D-Karte einbettet. Wichtig:

Overlay-Layer für unsere Akzente: User-Dot mit context aura, Back Factory Bloom, Ghost-Pulses
Falls die 3D-Karte als iframe kommt: positioned absolute, eigenes Pointer-Events Management
Falls native Three.js: shared canvas, wir mounten unsere Marker in dieselbe Szene
Color-grading layer obendrüber: ein leichter warmer multiply-blend layer (mix-blend-mode: multiply, #FFE8D1 mit 15% Opazität), damit die Karte tonal zur App passt

<MapPulse /> (Overlay-Marker)
Drei Varianten:

user: amber dot 12px, mit 60px context aura, 2s pulse
active-offer: amber bloom über Merchant, größer, 3s pulse, leichte Glow
ghost: gedämpft, anonymous pulse mit kleinem Timestamp-Label "vor 4 Min", langsam fadet

<CheckoutSheet /> (Wow Moment 3 — Highest Polish)
Mehrstufige Animation, full-screen sheet:

Stage 2 (1.2-5s): QR Code in Glasrahmen, Offer-Summary darunter, "Zum Scannen vorzeigen" Hint, dezenter "Simulate Scan" Button für Demo

Stage 5 (8-10s): Confirmation message in Serif: "Genieß es. Back Factory hat eine ruhige Stunde gefüllt."

Kreisförmiger Stempel, ~80px Durchmesser. Print-Feel: leicht raue Textur, gedämpfte Farbe pro Merchant, Datum unten klein in Mono. Props: merchant, date, state (collected | new | locked). State new glüht in Amber, scale 1.05 mit pulse.
<JournalEntry />
Horizontale Card im Journal. Layout: Datum links (Mono), Wetter-Emoji, Merchant-Name (Sans), gekauft (klein gedimmt), gespart (Amber, Mono). Sehr ruhig, listenartig.

<TransactionDensityChart /> (Merchant-Seite)
Line chart über den Tag (8:00-22:00). Die Zeit zwischen 13:00-14:00 ist visuell hervorgehoben — eine sanfte amber Highlight-Box mit Annotation "Quiet hour detected — KI aktiviert." Library: Recharts oder ein simples SVG. Style: keine Achsen-Boxen, dünne Linien, warmer Akzent für die aktuelle Stunde.
<FloatingNav />
Bottom-Nav als floating glass pill, ~16px vom unteren Rand. 4 Icons: Map, Passport, Journal, Settings. Aktiver State: amber Dot unter dem Icon, Icon selbst etwas kräftiger. Nicht-aktiv: niedriger Kontrast.

6. INTEGRATION MIT BACKEND & 3D-KARTE
Backend-Integration
Wahrscheinlich existieren Endpoints wie:

GET /api/context → aktuelles Wetter, Zeit, Location
GET /api/offers/current → aktives Angebot für User
POST /api/offers/:id/accept → Akzeptanz
GET /api/merchant/:id/dashboard → Dashboard-Daten
GET /api/user/passport → Stempel
GET /api/user/journal → Past Moments

Frontend-Strategie:

Zentrale lib/api.ts mit typed fetchers
React Query (TanStack Query) für Caching und Loading-States
Loading-States: niemals Spinner — stattdessen die Cards bloomen einfach später ein. Skeleton-States in extrem dezentem Glas-Look.
Error-States: warmer, ruhiger Ton — "Die Stadt holt kurz Luft." mit Retry-Button. Niemals rote Error-Banner.

Falls dein Freund noch keine Endpoints definiert hat: arbeitet zuerst ein API-Contract in einem api-contract.md aus, bevor das Frontend implementiert wird.
3D-Karten-Integration
Drei mögliche Szenarien — abhängig davon, was dein Freund gebaut hat:
Szenario A: Three.js / React Three Fiber Komponente

Direkt als React-Komponente importieren in <AmbientMap />
Frontend mountet eigene Marker (User-Dot, Bloom, Ghost-Pulses) in dieselbe Three.js Scene via shared ref
Color-grading via Post-Processing Layer in der Scene oder via CSS overlay

Szenario B: Spline-Export oder iframe

Als iframe in <AmbientMap /> einbetten
Marker als HTML overlay absolute positioniert über dem iframe
Position-Mapping: vom 3D-Map auf 2D-Coords via simple lookup table für die ~10 Merchant-Locations
Color-grading via overlay div mit mix-blend-mode: multiply

Szenario C: Mapbox GL mit 3D-Buildings

Mapbox-Komponente direkt mit mapbox-gl oder react-map-gl
Eigene Sources & Layers für Marker
Custom style von Mapbox Studio bauen, der zur warmen Palette passt — das ist wichtig, der Default-Style wird die Ästhetik zerstören

Empfehlung an deinen Freund: wenn die Karte bereits in einem dieser Formate existiert, lieber an die App-Ästhetik anpassen als andersrum. Der Color-Grade-Overlay ist ein guter Notnagel falls die Karte tonal nicht passt.

7. ASSETS & ILLUSTRATIONEN
Hero-Illustration im MomentCard (Brezel + Apfelschorle):

Empfehlung: Mit Midjourney oder DALL-E generieren mit Prompt wie: "warm minimalist illustration of a frosted glass of cold apple juice and a fresh german pretzel, soft peach and amber palette, ambient lighting, no background, slightly painterly, editorial style"
